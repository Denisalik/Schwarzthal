A take-home assignment for front-end developer position

We create a platform to tackle financial crime using AI. Behind a financial crime scheme there is never only one person or individual, but an organized network of companies and individuals. The platform is using machine learning algorithms that gather global multilingual data including companies’ ownerships, adverse media and sanction lists and allows to visualize, assess and analyze the network and the underlying risk of a client. A person whom we are looking for will work on the improvement of the platform.

The assignment description

You will need to create a single page application to search for content within the iTunes Store and explore activity of favorite artists. The application should have a few sections that allow to search music by its name, see the full list of favorite music and explore albums for the most favorite artists on a graph. The detailed description of each section will be provided further.

To search for content within the iTunes Store you should use iTunes Search API. You will need to create a few queries. The search query should return all songs (and nothing else except songs) for the search string. The lookup query should return all albums of a particular artist by the artist ID.

Technology stack

  • The application should be written in TypeScript (or JavaScript) and based on React framework
  • For graph visualization vis-network is strongly recommended

Requirements

There is no specific requirements on the appearance of the application. The sections design and layout is up to you, although visual appearance is important, try to do not spend a lot of time on it. Nice animations are welcome.

Search

The section should have a search box that allows to search music within the iTunes Store. The search result should be displayed below the search box. Pagination of the search result is not required, you are allowed to choose an appropriate limit of the displayed results by yourself. For each displayed song you should add a like button (or anything else that can be used for the same purpose). When a user click on the button, the liked song should be saved in the user history.

  • Search by song name;
  • Possibility to like/unlike favorite songs;
  • The user history is not required to be persistent across reloads;
History

The section should display all songs that the user ever liked and were added to the user history. The user history should be sorted in the historical order and either presented entirely on the screen or there should be implemented pagination that allows to load the songs incrementally. The history section should also provide a way to remove a song from the favorites through like/unlike functionality. After a song was unlicked, it should disappear.

  • Display user history;
  • Possibility to remove favorite songs;
Statistics

On this section should be displayed the top 5 most favorite artists of the user. The top 5 artists should be derived from the user history by counting the number of songs of artists. The statistics should be presented on a graph where artists are connected to released albums, all represented as nodes. The initial position of nodes that represent an artist should be fixed and such nodes should not be possible to move. Note that sometimes artists publish a single as an album, so to not pollute the graph make sure the album is really an album and has at least 2 songs.

  • Display on the graph the most favorite artists and released by them albums;
  • The artists nodes should be fixed on the graph;
